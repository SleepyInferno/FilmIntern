### Documentation Discrepancy (Low Severity)

`docs/unraid-deployment.md` line 134 documents the health check response as `{"status":"ok","version":"...","database":"connected"}` but the actual `src/app/api/health/route.ts` returns `{"status":"ok","version":"...","db":"connected"}` — the key is `"db"`, not `"database"`. Runtime behavior is unaffected (Docker and Caddy evaluate only HTTP 200). Affects: DOCK-05, CI-05.

### Flow 2: Push to main → Unraid Deployment (CI-01 through CI-05, PROD-01, PROD-02, DOCK-01 through DOCK-06)

**Status: COMPLETE**

```
git push main
  → docker-publish.yml triggers
  → BuildKit builds Dockerfile (bookworm-slim, standalone, non-root nextjs uid 1001)
  → GHA layer cache (cache-from/to: type=gha,mode=max)
  → push ghcr.io/sleepyinferno/filmintern:sha-<SHA> + :latest
  → Unraid operator: docker login → chown -R 1001:1001 data dir
  → docker compose -f docker-compose.prod.yml up -d
  → app container: HEALTHCHECK probes /api/health every 30s (40s start-period)
  → caddy starts after app healthy (condition: service_healthy)
  → Caddy :7430 → reverse_proxy app:3000 with flush_interval -1 (SSE streaming)
  → UI accessible at http://UNRAID_IP:7430
```
