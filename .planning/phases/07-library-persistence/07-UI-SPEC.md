## Layout Contract

### Filter Panel (inside ProjectsSidebar)

- Position: between the sidebar header ("Projects" + "+" button) and the project list
- Container: `div` with `px-4 py-2 border-b border-border`
- Layout: vertical flex column, `gap-1`
- Section label: "Filter" in text-xs font-semibold text-muted-foreground uppercase tracking-wider (matches existing "Projects" header style)
- Each filter row: horizontal flex, `items-center gap-2`
- Each row contains: native HTML checkbox input (styled via accent-color or Tailwind forms) + label text

### Filter Checkboxes

- "All" is a checkbox at the top that toggles all types on/off
- When all individual types are checked, "All" shows as checked
- When any individual type is unchecked, "All" shows as unchecked
- Unchecking "All" unchecks all types (shows no projects -- edge case; immediately re-checks all to prevent empty state)
- Checking "All" checks all types

**Checkbox labels (matching PROJECT_TYPES registry):**

| Checkbox Label | Filter Key | Icon |
|---------------|------------|------|
| All | (meta) | none |
| Narrative | `narrative` | Film (13px) |
| Documentary | `documentary` | Video (13px) |
| Corporate | `corporate` | Briefcase (13px) |
| TV / Episodic | `tv-episodic` | Tv (13px) |

Note: Short-form project type does not exist in the codebase (`PROJECT_TYPES` registry has only 4 types). Do not include a short-form filter checkbox.

### Filter Behavior

- Filter is client-side only -- applied to the already-fetched `projects` array via `.filter()`
- Filter state: `Set<string>` of checked project type keys, initialized with all 4 types
- Filter state stored in component-local `useState` (session-only, not persisted to localStorage)
- When no projects match the active filter, show the existing empty state: FolderOpen icon + "No projects yet"

### Project List (existing, unchanged)

- Layout: vertical list inside `overflow-y-auto` container
- Each item: `px-3 py-2.5`, flex column with `gap-0.5` (see Spacing Scale exceptions)
- First row: title (text-sm font-medium truncate) + delete button (opacity-0, group-hover:opacity-60)
- Second row: type icon + type label + dot separator + time-ago (all text-xs text-muted-foreground)
- Active project: `bg-accent` background
- Hover: `hover:bg-accent`
- Delete button hover: `hover:text-destructive`

## Interaction Contract

### Auto-Save (invisible)

| Trigger | Action |
|---------|--------|
| `isStreaming` transitions from `true` to `false` AND `analysisData` is non-null | Call `saveAnalysis()` silently -- no toast, no indicator |
| User generates a derivative document (treatment/outline) | Call `updateProject()` to persist the new document alongside existing analysis |
| Re-analysis on same project | Call `updateProject()` to overwrite -- no duplicate entries |
| Save failure | Silent failure -- console.error only, no user-facing error |

### Filter Checkboxes

| Trigger | Action |
|---------|--------|
| Check/uncheck individual type | Toggle that type in filter Set, re-filter project list |
| Check "All" | Set filter to all 4 types |
| Uncheck "All" | Set filter to all 4 types (prevent empty filter -- "All" unchecked re-checks everything) |
| All individual types manually checked | "All" checkbox visually checks (derived state) |
| Any individual type unchecked | "All" checkbox visually unchecks (derived state) |

### Open from Library (existing, unchanged)

| Trigger | Action |
|---------|--------|
| Click project in sidebar | `loadProject(id)` loads `analysisData`, `reportDocument`, `title`, `projectType` into workspace |
| Navigate to non-home page | Router pushes to `/` after loading |

### Delete from Library (existing, unchanged)

| Trigger | Action |
|---------|--------|
| Click trash icon on project row | Immediate delete via `DELETE /api/projects/:id` -- no confirmation dialog |
| Delete active project | Workspace resets via `resetWorkspace()` |
