## Page Layout Contract

### Revision Page Shell (`/revision/[projectId]`)

```
+----------------------------------------------------------+
| AppTopNav  (h-14, "Film Intern" brand)                   |
+----------+-----------------------------------------------+
| Projects | Main Content (max-w-[1100px], px-8, py-6)     |
| Sidebar  |                                                |
|          |  [<- Back]  Project Title                      |
|          |              Project Type Badge                |
|          |                                                |
|          |  +------------------------------------------+  |
|          |  | Card: Suggestion Generation Placeholder  |  |
|          |  | (py-12, centered muted text)             |  |
|          |  +------------------------------------------+  |
|          |                                                |
|          |  +------------------------------------------+  |
|          |  | Card: Review & Export Placeholder         |  |
|          |  | (py-12, centered muted text)             |  |
|          |  +------------------------------------------+  |
+----------+-----------------------------------------------+
```

Layout inherits the root layout (`layout.tsx`): `AppTopNav` + `ProjectsSidebar` + constrained `max-w-[1100px]` main area. No layout overrides for Phase 15.

### Loading State

While project data loads from `/api/projects/[projectId]`:
- Show `Skeleton` bars: one at 200px width for title, one at 100px for type badge
- Show two `Skeleton` cards at full width, height 120px each
- Duration: typically < 500ms (local API call)

### Empty State (no analysis data)

When `analysisData` is null on the loaded project:
- Center-aligned message in the main content area
- No placeholder cards shown

### Navigation Button on Workspace

Added inside the completed analysis block of `page.tsx` (around line 474), alongside existing UI:
- `outline` variant button, `sm` size
- Positioned in the project info bar area
- Only visible when `reportDocument` exists and `!isAnalyzing`

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click "Adjustments & Revision" on workspace | Navigate to `/revision/{projectId}` via `next/link` (client-side navigation) |
| Click back arrow on revision page | Navigate to `/` via `next/link` |
| Direct URL access `/revision/{id}` | Load project data from API independently (no workspace context dependency) |
| Direct URL with invalid project ID | Show error state with link back to workspace |
| Direct URL with project missing analysis | Show empty state ("Analysis not yet completed") |

No modals, no toasts, no form inputs, no animations beyond default Tailwind transitions.
