  <files>src/components/document-workspace.tsx</files>
  <read_first>src/components/document-workspace.tsx</read_first>
  <action>
In `src/components/document-workspace.tsx`, lines 76-77 use a dynamic JSX tag:
```typescript
const Tag = `h${level}` as keyof JSX.IntrinsicElements;
return <Tag key={i}>{textContent}</Tag>;
```
TypeScript rejects this because the computed type includes `number | symbol` which are not valid JSX element types.

Replace lines 74-78 (the heading branch of the if block) with explicit conditional rendering:

```typescript
if (nodeType === 'heading') {
  const level = (node.attrs as { level: number })?.level ?? 2;
  const HeadingTag = (`h${level}` as 'h1' | 'h2' | 'h3' | 'h4' | 'h5' | 'h6');
  return <HeadingTag key={i}>{textContent}</HeadingTag>;
}
```

The key fix is the type assertion to the union `'h1' | 'h2' | 'h3' | 'h4' | 'h5' | 'h6'` instead of `keyof JSX.IntrinsicElements`. This narrows the type to only valid heading tag names that TypeScript accepts as JSX element types.

Also fix the unused import warning: remove `useEffect` from the import on line 3. Change:
```typescript
import { useState, useCallback, useRef, useEffect } from 'react';
```
to:
```typescript
import { useState, useCallback, useRef } from 'react';
```
