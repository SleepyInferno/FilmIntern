Create directory `.github/workflows/` and the file `docker-publish.yml` with the following exact content:

```yaml
name: Build and Publish Docker Image

on:
  push:
    branches: [main]

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v4

      - name: Log in to GitHub Container Registry
        uses: docker/login-action@v4
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract metadata (tags, labels)
        id: meta
        uses: docker/metadata-action@v6
        with:
          images: ghcr.io/${{ github.repository }}
          tags: |
            type=sha
            type=raw,value=latest,enable={{is_default_branch}}

      - name: Build and push
        uses: docker/build-push-action@v7
        with:
          context: .
          push: true
          platforms: linux/amd64
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

Key decisions embedded in the workflow:
- **permissions block** (CI-01): `packages: write` is required or GHCR push fails with "permission_denied". `contents: read` is explicit best practice.
- **setup-buildx-action** (CI-02): Required for `type=gha` cache backend. No QEMU action needed -- single-platform build.
- **cache-to mode=max** (CI-02): Caches ALL intermediate layers including deps and builder stages. The default `mode=min` only caches the final layer, which would miss the expensive `npm ci` and `npm run build` stages in the multi-stage Dockerfile.
- **metadata-action with type=sha** (CI-03): Produces tags like `sha-90dd603` (7-char short SHA). Combined with `type=raw,value=latest,enable={{is_default_branch}}`, each push to main produces exactly two tags.
- **platforms: linux/amd64** (CI-04): Single platform -- no QEMU cross-compilation needed. Matches target environments (Docker Desktop on Windows/Mac, Unraid x86-64).
- **github.repository** in images: The metadata-action automatically lowercases this, avoiding the GHCR "invalid reference format" pitfall with mixed-case repo names like `SleepyInferno/FilmIntern`.

Do NOT include:
- `docker/setup-qemu-action` -- not needed for single-platform amd64 builds, adds ~15s for no benefit
- `type=ref,event=branch` tag -- this creates a tag named "main", not "latest"
- Any PAT-based auth -- GITHUB_TOKEN with packages:write is sufficient for same-repo GHCR
