<decisions_made>

- `.optional()` → `.nullable()` (not `.nullish()`) — keeps fields in JSON Schema `required` while allowing null; all workspace guards use `&&` or `??` which handle null correctly
- `gpt-5.4` classified as reasoning model by `@ai-sdk/openai@3.0.41` → routed to `/v1/responses` (Responses API), not `/v1/chat/completions`
- Source text capped at 6000 chars in doc gen prompts — analysis JSON also in context, need token headroom
- Prompt Tiptap node examples are inline JSON (not prose descriptions) — more reliable for model to reproduce exactly
- analysis_prompts/ pack is 2-pass; Pass 2 (prose rendering) redundant for card UI — only Pass 1 integrated
- `overallScore` (numeric) kept alongside `verdict` (verbal) — not removed
