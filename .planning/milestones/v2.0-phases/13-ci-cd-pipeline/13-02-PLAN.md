**Title:** `# Deploying FilmIntern on Unraid`

**Section: Prerequisites**
- Unraid 6.12+ with Docker enabled
- GitHub account with access to the SleepyInferno/FilmIntern repository
- GitHub Personal Access Token (PAT) with `read:packages` scope
  - Created at: Settings > Developer settings > Personal access tokens > Tokens (classic)
  - Only the `read:packages` scope is needed

**Section: Step 1 -- Authenticate to GHCR**
```bash
echo "YOUR_GITHUB_PAT" | docker login ghcr.io -u YOUR_GITHUB_USERNAME --password-stdin
```
- Explain: This stores credentials in `/root/.docker/config.json` on Unraid. Only needed once (persists across reboots).
- Note: Replace `YOUR_GITHUB_PAT` and `YOUR_GITHUB_USERNAME` with actual values.

**Section: Step 2 -- Create data directory**
```bash
mkdir -p /mnt/user/appdata/filmintern
chown -R 1001:1001 /mnt/user/appdata/filmintern
```
- Explain: The container runs as uid 1001 (non-root). The data directory stores the SQLite database and settings. The `chown` ensures the container user can write to it.

**Section: Step 3 -- Create deployment files**
- Instruct user to create a directory (e.g., `/mnt/user/appdata/filmintern-config/`) and place two files:
  1. `docker-compose.prod.yml` -- include the FULL contents of the production compose file (copy from the repo's `docker-compose.prod.yml`)
  2. `Caddyfile` -- include the FULL contents of the Caddyfile (copy from the repo's `Caddyfile`)
- Note: These files are also available in the GitHub repository at the project root.

**Section: Step 4 -- Deploy**
```bash
cd /mnt/user/appdata/filmintern-config
docker compose -f docker-compose.prod.yml up -d
```
- Explain: First run pulls the image from GHCR (~440MB) and starts both the app and Caddy reverse proxy.

**Section: Step 5 -- Verify**
```bash
# Check containers are running
docker compose -f docker-compose.prod.yml ps

# Check health endpoint (wait ~40s for startup)
curl http://localhost:7430/api/health
```
- Expected output: JSON with `{"status":"ok","version":"...","database":"connected"}`
- Access the UI at `http://YOUR_UNRAID_IP:7430`

**Section: Updating to Latest Version**
```bash
cd /mnt/user/appdata/filmintern-config
docker compose -f docker-compose.prod.yml pull
docker compose -f docker-compose.prod.yml up -d
```
- Explain: `pull` fetches the latest image from GHCR. `up -d` recreates the container with the new image. Data persists in the volume.

**Section: Rolling Back to a Previous Version**
```bash
# List available versions
# Visit: https://github.com/SleepyInferno/FilmIntern/pkgs/container/filmintern/versions
# Or use: docker pull ghcr.io/sleepyinferno/filmintern:sha-XXXXXXX

# Edit docker-compose.prod.yml to pin a specific SHA tag:
# image: ghcr.io/sleepyinferno/filmintern:sha-90dd603

docker compose -f docker-compose.prod.yml up -d
```
- Explain: Each build produces a SHA-tagged image. Pin to a specific SHA to roll back. Change back to `:latest` to resume tracking the latest build.

**Section: Troubleshooting**

| Problem | Cause | Fix |
|---------|-------|-----|
| `unauthorized` on pull | GHCR auth expired or missing | Re-run `docker login ghcr.io` with PAT |
| `permission denied` on data dir | Wrong ownership | `chown -R 1001:1001 /mnt/user/appdata/filmintern` |
| Health check fails | App still starting | Wait 40s after container start, check logs with `docker compose logs app` |
| Ollama not reachable | Missing host networking | Verify `extra_hosts` in compose file; on Linux Docker use `host.docker.internal:host-gateway` |

**Section: Configuration**

| Environment Variable | Default | Description |
|---------------------|---------|-------------|
| `DATABASE_PATH` | `/app/data/filmintern.db` | SQLite database file location |
| `SETTINGS_DIR` | `/app/data/.filmintern` | Application settings directory |

AI provider API keys can be configured through the Settings UI after first launch.

Do NOT include:
- Kubernetes or Helm instructions (out of scope)
- Docker Hub references (GHCR only)
- Multi-architecture instructions (amd64 only)
- Community Applications XML template submission (personal use, markdown guide sufficient)
