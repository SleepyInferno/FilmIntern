Create docker-compose.yml with this exact content:

```yaml
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile.dev
    ports:
      - "3000:3000"
    volumes:
      - .:/app
      - /app/node_modules
      - /app/.next
      - ./data:/app/data
    environment:
      - WATCHPACK_POLLING=true
      - TURBOPACK=0
      - DATABASE_PATH=/app/data/filmintern.db
      - SETTINGS_DIR=/app/data/.filmintern
      - OLLAMA_BASE_URL=http://host.docker.internal:11434/api
    env_file:
      - path: .env
        required: false
    extra_hosts:
      - "host.docker.internal:host-gateway"
```

Key details:
- TURBOPACK=0: Phase 10 discovered Turbopack incompatibility in Docker; use Webpack for reliability
- WATCHPACK_POLLING=true: Required for HMR on Windows/WSL2 where inotify events don't propagate through bind mounts
- Anonymous volumes for /app/node_modules and /app/.next: Isolate container-specific artifacts from host bind mount
- ./data:/app/data: Persist SQLite database across container restarts
- DATABASE_PATH=/app/data/filmintern.db and SETTINGS_DIR=/app/data/.filmintern: Match the env var names established in Phase 10 (db.ts line 4, settings.ts line 18)
- OLLAMA_BASE_URL: Will be consumed by settings.ts after Plan 02 adds support
- env_file with required: false: Compose v2.24+ feature allowing startup without .env file
- extra_hosts: Maps host.docker.internal to host gateway IP for Ollama connectivity
