Create two files at the project root:

**Caddyfile:**
```caddyfile
# Production reverse proxy for FilmIntern
# Listens on port 7430 (HTTP only, LAN access)
# To enable HTTPS later: replace ":7430" with your domain name

:7430 {
	reverse_proxy app:3000 {
		flush_interval -1
	}
}
```
Key points:
- `:7430` binds all interfaces, HTTP only (no domain = no automatic HTTPS per user decision)
- `app` resolves via Docker Compose internal DNS (service name)
- `flush_interval -1` disables response buffering -- required for streaming AI analysis routes (`/api/analyze`, `/api/analyze/critic`) which use `toTextStreamResponse()`
- Do NOT add `encode gzip` -- known Caddy issue #6293 where compression buffers streaming responses, breaking real-time delivery. LAN-only deployment means no bandwidth concern.

**docker-compose.prod.yml:**
```yaml
# Production deployment for Unraid Docker Compose Manager
# Prerequisites:
#   1. Authenticate to GHCR (private repo):
#      echo "YOUR_GITHUB_PAT" | docker login ghcr.io -u sleepyinferno --password-stdin
#      (PAT needs read:packages scope)
#   2. Create and own the data directory:
#      mkdir -p /mnt/user/appdata/filmintern && chown -R 1001:1001 /mnt/user/appdata/filmintern

services:
  app:
    image: ghcr.io/sleepyinferno/filmintern:latest
    volumes:
      - /mnt/user/appdata/filmintern:/app/data
    environment:
      - DATABASE_PATH=/app/data/filmintern.db
      - SETTINGS_DIR=/app/data/.filmintern
    extra_hosts:
      - "host.docker.internal:host-gateway"
    restart: unless-stopped
    networks:
      - internal

  caddy:
    image: caddy:2-alpine
    ports:
      - "7430:7430"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile:ro
      - caddy_data:/data
    depends_on:
      app:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - internal

volumes:
  caddy_data:

networks:
  internal:
```

Key points:
- App container has NO `ports` directive -- only reachable through Caddy (per user decision)
- `depends_on` with `condition: service_healthy` uses the existing HEALTHCHECK in the Dockerfile (curl to `/api/health`)
- `extra_hosts` with `host-gateway` enables Ollama host connectivity on Linux/Unraid (matches dev compose pattern)
- Caddyfile mounted read-only (`:ro`) into Caddy container at `/etc/caddy/Caddyfile`
- `caddy_data` named volume for Caddy internal state (cert storage, config cache) -- NOT user data
- Named network `internal` keeps services isolated and resolvable by service name
- `restart: unless-stopped` on both services for production reliability
- Comments at top document the two prerequisite steps (GHCR auth and directory permissions) -- this IS the setup documentation since there is no separate doc phase
