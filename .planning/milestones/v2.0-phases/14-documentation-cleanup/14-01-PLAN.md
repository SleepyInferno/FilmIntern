# Plan 14-01: Documentation Cleanup + Prod Ollama Fix

## Context

Phase 12 verification flagged two documentation inconsistencies:
1. REQUIREMENTS.md and ROADMAP.md still described PROD-01 as "automatic HTTPS" despite the locked HTTP-only implementation
2. `docs/unraid-deployment.md` health response example used key `"database"` while the actual `/api/health` returns `"db"`

The v2.0 milestone audit additionally flagged:
3. `docker-compose.prod.yml` missing `OLLAMA_BASE_URL` — `extra_hosts` wires the network path but the app defaults to `http://localhost:11434` (container loopback), not the host. Production Ollama users must know to set this.
