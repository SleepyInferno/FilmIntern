  <name>Task 1: Verify all project types produce correct analysis end-to-end</name>
  <what-built>
All 5 project types (Documentary, Corporate Interview, Narrative Film, TV/Episodic, Short-form/Branded) now have:
- Type-specific Zod schemas defining the analysis data shape
- Domain-expert system prompts driving Claude's analytical output
- Dedicated report components displaying the analysis
- API routing dispatching the correct schema+prompt per project type
- Page-level routing to the correct report component
- Short-form has a secondary input type toggle (Script/Storyboard | VO Transcript | Rough Outline)
- Narrative and TV/Episodic reports use internal tabs (Structure|Coverage and Episode Arc|Series Structure)
  </what-built>
  <how-to-verify>
1. Start the dev server: `npm run dev`
2. Open http://localhost:3000

**Test Documentary (existing, regression check):**
- Select "Documentary" tab
- Upload a text file with interview content
- Click "Run Analysis"
- Verify: Report shows Summary, Key Quotes, Recurring Themes, Key Moments, Editorial Notes

**Test Corporate Interview:**
- Select "Corporate Interview" tab
- Upload a text file with corporate interview content (e.g., a CEO talking about company vision)
- Click "Run Analysis"
- Verify: Report shows Summary (with context type badge), Soundbites (with usability color coding), Messaging Themes (with consistency badges), Speaker Effectiveness (with quotability/on-message badges), Editorial Notes
- Verify: Output uses corporate vocabulary (key messaging, brand voice, soundbite) not generic AI summary language

**Test Narrative Film:**
- Select "Narrative Film" tab
- Upload a screenplay or script (PDF or FDX)
- Click "Run Analysis"
- Verify: Report shows two tabs: "Structure" and "Coverage"
- Structure tab: Story Beats (with named beats and effectiveness badges), Pacing and Tension, Structural Strengths/Weaknesses
- Coverage tab: Characters (with role badges and arc assessments), Conflict, Dialogue, Marketability (comp titles, logline), Overall Assessment
- Verify: Output uses screenplay vocabulary (logline, turning point, act break)

**Test TV/Episodic:**
- Select "TV / Episodic" tab
- Upload a TV pilot script
- Click "Run Analysis"
- Verify: Report shows two tabs: "Episode Arc" and "Series Structure"
- Episode Arc tab: Cold Open, Story Strands (A/B/C), Character Introductions, Episode Arc
- Series Structure tab: Premise Longevity, Serialized Hooks, Episodic vs Serial Balance, Season Arc Potential
- Verify: Output uses TV vocabulary (cold open, runner, serialized hooks)

**Test Short-form/Branded:**
- Select "Short-form / Branded" tab
- Verify: Input type toggle appears below project type tabs with 3 options
- Select "VO Transcript", upload a commercial VO transcript
- Click "Run Analysis"
- Verify: Report shows Hook Strength, Pacing, Messaging Clarity, CTA Effectiveness, Emotional/Rational Balance
- Switch input type to "Script / Storyboard" and re-run -- verify analysis lens shifts appropriately

**Test Type Switching:**
- Run analysis on one type, then switch to a different type
- Verify: Previous analysis data clears (no stale data from wrong type)
- Verify: No console errors during transition

**Test Streaming:**
- During any analysis run, verify report sections appear progressively (not all at once)
- Verify: Skeleton placeholders show for sections not yet received
